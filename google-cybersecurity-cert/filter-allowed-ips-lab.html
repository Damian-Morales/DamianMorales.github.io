Filtering Allowed IPs from Text Files – My Lab

Overview

In this lab, I wrote a Python script to parse a list of permitted IP addresses from allow_list.txt, then automatically remove any addresses listed in the provided remove_list variable. This helped me ensure that only current, authorized IPs remain on file.

Scenario

As a security analyst, I needed to automate maintenance of our access-control list. I was given:

  • allow_list.txt containing all IPs currently allowed to reach sensitive systems.
  • A Python list named remove_list holding IPs that should no longer have access.

My task was to read both sources, filter out the unwanted addresses, and rewrite the cleaned list back to the file.

What I Did

  • Opened allow_list.txt in read mode and used .read().split() to load its IPs into a list.
  • Iterated through that list with a for loop, using an if check to skip any IPs found in remove_list.
  • Reopened allow_list.txt in write mode and used .write() to save only the remaining IPs, one per line.
  • Wrapped all file operations in with blocks to guarantee proper resource cleanup.

Conclusion

  • with blocks ensure files are closed automatically.
  • Reading with .read().split() converts file lines into a Python list.
  • Using a for loop plus an if test lets me filter out unwanted entries.
  • Writing with .write() in "w" mode cleanly replaces the old file contents.
  • Algorithms like this can be packaged into functions for reuse.
