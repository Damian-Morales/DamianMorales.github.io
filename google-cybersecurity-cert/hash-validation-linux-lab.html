Validate File Integrity with Hashes

Overview

As a security analyst, youâ€™ll need to implement security controls to protect organizations against a variety of threats. This lab focused on hashing, an important method for verifying file integrity.

A hash is a unique string generated by an algorithm that reflects the contents of a file. If the file changes, even slightly, its hash will change. This makes hashing a reliable method for detecting tampering or unauthorized changes.

In this lab, I computed hashes for two files and used Linux commands to compare them. This helped simulate how integrity validation can prevent the spread of malicious or corrupted data.

Scenario

I was tasked with determining whether two files were identical or different by generating and comparing their hashes using command-line tools.

First, I examined the content of the two files. Next, I generated hash values for both files using a hashing utility. Finally, I manually compared the hash outputs to determine file similarity.

Note: The lab began with me logged in as user analyst, and the home directory set to /home/analyst.

Conclusion

This lab introduced core Linux tools for validating file integrity. I learned how to:

  • Generate hashes with sha256sum
  • Display contents with the cat command
  • Compare files using the cmp command

These techniques are useful for confirming that data has not been modified or corrupted and play an important role in verifying software and security logs.
