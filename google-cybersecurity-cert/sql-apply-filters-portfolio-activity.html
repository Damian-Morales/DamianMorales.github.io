Apply Filters to SQL Queries

Project Description

This activity demonstrates my ability to write SQL queries that apply multiple filtering techniques to investigate potential security issues. I used filters such as AND, OR, NOT, and LIKE across two datasets: log_in_attempts and employees.

Queries and Explanations

1. After Hours Failed Logins

SELECT * FROM log_in_attempts WHERE login_time > '18:00' AND success = 0;

This query filters for all failed login attempts after 6:00 PM, combining time and success conditions using AND.

2. Login Attempts on Specific Dates

SELECT * FROM log_in_attempts WHERE login_date = '2022-05-08' OR login_date = '2022-05-09';

This query uses OR to return login attempts from both May 8 and May 9, 2022.

3. Logins Outside of Mexico

SELECT * FROM log_in_attempts WHERE country NOT LIKE 'MEX%';

This query uses NOT and LIKE to exclude logins from Mexico and find foreign attempts.

4. Marketing Employees in East Offices

SELECT * FROM employees WHERE department = 'Marketing' AND office LIKE 'East%';

This query filters for Marketing employees in East office locations using AND and pattern matching with LIKE.

5. Employees in Finance or Sales

SELECT * FROM employees WHERE department = 'Finance' OR department = 'Sales';

This filter returns all employees in either Finance or Sales departments using OR.

6. Employees Not in IT

SELECT * FROM employees WHERE NOT department = 'Information Technology';

This final query filters out the IT department to return all other employees using !=.

Summary

This portfolio activity demonstrates practical knowledge of filtering SQL queries using AND, OR, NOT, LIKE, and date/time conditions. These queries reflect how security analysts detect suspicious behavior and investigate access patterns using structured query logic.
