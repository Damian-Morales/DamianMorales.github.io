Apply Filters to SQL Queries

Project Description

This activity demonstrates my ability to write SQL queries that apply multiple filtering techniques to investigate potential security issues. I used filters such as AND, OR, NOT, and LIKE across two datasets: log_in_attempts and employees.

Queries and Explanations

1. After Hours Failed Logins

SELECT * FROM log_in_attempts WHERE login_time > '18:00' AND success = 0;

This query filters for all failed login attempts that happened after 6:00 PM. It uses AND to combine time and success conditions.

2. Login Attempts on Specific Dates

SELECT * FROM log_in_attempts WHERE login_date = '2022-05-08' OR login_date = '2022-05-09';

This query uses OR to retrieve logins from two specific dates, allowing investigation of a security incident timeline.

3. Logins Outside of Mexico

SELECT * FROM log_in_attempts WHERE country NOT LIKE 'MEX%';

This uses NOT with LIKE to exclude entries from Mexico, capturing foreign login attempts.

4. Marketing Employees in East Offices

SELECT * FROM employees WHERE department = 'Marketing' AND office LIKE 'East%';

This query filters for Marketing employees who are located in offices that begin with "East", using both AND and LIKE.

5. Employees in Finance or Sales

SELECT * FROM employees WHERE department = 'Finance' OR department = 'Sales';

This simple OR filter identifies employees in either Finance or Sales, often grouped for administrative reviews.

6. Employees Not in IT

SELECT * FROM employees WHERE department != 'Information Technology';

This query retrieves all employees except those in IT. This is useful when targeting departments that still require updates.

Summary

This lab helped reinforce my understanding of SQL filtering using AND, OR, NOT, LIKE, and specific date/time conditions. These skills apply to real-world threat analysis, data filtering, and compliance investigations in a security role.
