DNS & ICMP Incident Analysis Report

Overview

This report examines DNS query failures and ICMP unreachable packets observed in network traffic. The analysis is based on data from a lab environment simulating unusual DNS behavior and network interruptions. The full traffic log is available here: DNS/ICMP Traffic Log .

Scenario Summary

DNS requests from internal hosts consistently failed to resolve a specific domain. In response, ICMP Destination Unreachable messages were generated, indicating that the DNS servers were unreachable. The pattern suggested either misconfigured routing or a denial-of-service event affecting the availability of DNS services.

Key Findings

  • Multiple DNS requests with no response
  • ICMP Type 3 Code 1 (Destination Unreachable, Host Unreachable) repeatedly returned
  • Downstream communication with critical infrastructure was disrupted

Tools Used

  • Wireshark for protocol filtering and analysis
  • ICMP and DNS display filters
  • Packet follow-up analysis

Conclusion

The ICMP error messages confirmed that internal systems could not reach the DNS service. This failure caused interruptions to name resolution and reflected availability issues. The ability to identify these patterns through packet analysis helps detect and respond to infrastructure disruptions early.
