Overview

I assessed and hardened my home network by signing into the router’s admin portal, documenting the environment, and disabling risky features that commonly expand attack surface in consumer setups. The work produced a clear before/after record and a repeatable checklist for ongoing security.

0 inbound services exposed (WAN)
WPA2 verified on Wi-Fi
Logging enabled for visibility

Step 1 — Network Mapping

Logged into the router UI (192.168.1.1) and captured baseline details: model/firmware, wireless settings, connected devices, DHCP range.

Step 2 — Router Security Assessment

I reviewed key security settings to ensure strong defaults and no unnecessary exposure.

Step 3 — Hardening Actions

Applied changes to reduce attack surface and add visibility.

  • Enabled system logging
  • Verified firewall is active
  • Disabled Port Forwarding and DMZ
  • Reviewed connected devices for trust
  • Kept ISP-managed secure DNS
Result: No inbound WAN exposure, modern Wi-Fi encryption, and improved logging/visibility.

Conclusion & Recommendations

The router now follows secure defaults and least-privilege principles. Next steps: monthly firmware checks, periodic Wi-Fi password rotation, log review for anomalies, and (optionally) create a guest SSID for segmentation.
