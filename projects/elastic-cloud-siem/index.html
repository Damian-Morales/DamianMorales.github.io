Overview

In this mini-SIEM build, I deployed an Elastic Cloud deployment, enrolled a Windows host with Elastic Agent, ingested Windows Security/PowerShell logs, turned on prebuilt detection rules, and then generated benign PowerShell activity to validate end-to-end detection and investigation in Kibana. The test successfully produced alerts, which I explored using the Investigate in timeline workflow.

Objectives

  • Spin up Elastic Cloud & configure the Windows integration.
  • Enroll a Windows endpoint and confirm healthy telemetry.
  • Enable Elastic prebuilt rules related to authentication and PowerShell.
  • Generate test PowerShell events and confirm alerting.
  • Investigate alerts via host insights and Timeline.

Environment

  • Elastic Cloud (Kibana + Elasticsearch)
  • Elastic Agent on Windows with system and Windows integrations
  • Data streams: Windows Security, PowerShell/Script Block, Sysmon (where available)
